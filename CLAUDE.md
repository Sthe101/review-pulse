## Conventions (apply across steps)
- **Supabase narrow-projection cast**: supabase-js v2 chain types infer `never` for narrow `.select(col)` projections. Pattern: `(await ... .maybeSingle()) as { data: { col: T } | null }` or `as unknown as { update: (v) => { eq: (c, v) => Promise<{error}> } }`. Used in callback route, onboarding page, checklist updater, demo seeder.
- **Hoisted vi.mock pattern**: `const fnMock = vi.hoisted(() => vi.fn())` so the mock is available in `vi.mock()` factories. Standard for `pushMock` (next/navigation), Supabase auth/`from` chains, `toastErrorMock` (sonner).
- **Node test pragma**: `// @vitest-environment node` at top of files using `next/server` (`NextRequest`/`NextResponse`) — required since `vitest.config.ts` dropped `environmentMatchGlobs` (removed in Vitest 4).
- **Redirect status tolerance**: assert `[301, 302, 303, 307, 308]` rather than 307 exact, so tests don't break if Next changes default.
- **Auth-page Field component**: each auth page (`signup`, `login`, `forgot-password`) has its own local `Field` (icon prefix via absolute-positioned `<Icon>`, red border via `var(--neg)` on error, `aria-invalid`/`aria-describedby`). No shared component yet — revisit if a 4th auth page lands.
- **`EMAIL_REGEX = /^[^\s@]+@[^\s@]+\.[^\s@]+$/`** is the single email validation regex.
- **Safe error messaging**: never echo raw Supabase error strings in UI. Login → "Invalid email or password"; forgot-password → "Something went wrong. Please try again."; callback → short codes (`oauth`/`missing_code`/`auth_failed`) in URL.

### Step 4 — Auth middleware & routing
- `src/middleware.ts` — top-level Next middleware. Logic:
  - **Bypass prefixes** (`/api/stripe/webhook`, `/api/cron`, `/report`) → `NextResponse.next()` before any Supabase call (webhooks/cron must never be redirected or do `auth.getUser()` round-trip).
  - Otherwise call `updateSession(request)` (refreshes session cookie + returns user).
  - **Protected prefixes** (`/dashboard`, `/projects`, `/trends`, `/integrations`, `/settings`, `/billing`; exact + `/` subpaths): unauth → 302 `/login?next=<pathname>`.
  - **Auth pages** (`/login`, `/signup` exact): authed → 302 `/dashboard`.
  - On every redirect, copy refreshed cookies onto the redirect: `supabaseResponse.cookies.getAll().forEach(c => redirect.cookies.set(c))`.
  - `config.matcher` excludes `_next/static`, `_next/image`, `favicon.ico`, asset extensions (svg/png/jpg/jpeg/gif/webp/ico).
- Next 16 deprecation (non-blocking): `"middleware" file convention deprecated, use "proxy"`. Build labels it `ƒ Proxy (Middleware)`. Migration path: rename to `proxy.ts` + export `proxy`.
- Verified: build clean, `npm test` 1 passed.
